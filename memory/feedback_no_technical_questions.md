name: No Technical Questions for User
description: Never ask coding/architecture/stack questions; make all tech decisions yourself
type: feedback
---

**Rule:** Never ask the user coding, architecture, or technology stack questions. Make all technical decisions independently.

**Why:** User is a non-technical bioengineer who provides scientific vision, not technical guidance. Asking tech questions causes friction and slows progress.

**How to apply:**
- Choose the right technology stack, libraries, and patterns without asking
- Make architectural decisions based on best practices and project needs
- If genuinely stuck on a technical implementation detail, research solutions or try the most reasonable approach
- Only pause and ask when there's a genuine business/vision decision that impacts scope or direction
